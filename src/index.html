Java Standard Streams

Understanding System.in, System.out, and System.err

üì• System.in

This is the standard input stream. It's used to read data from an input source, typically the keyboard.

// Reading from System.in
Scanner scanner = new Scanner(System.in);
System.out.print("Enter your name: ");
String name = scanner.nextLine();

Default source: Keyboard

üì§ System.out

This is the standard output stream. It's used to write data to an output destination, typically the console.

// Writing to System.out
System.out.println("Hello, World!");
System.out.print("This is a message.");

Default destination: Console

‚ö†Ô∏è System.err

This is the standard error stream. It's used to output error messages, typically to the console.

// Writing to System.err
System.err.println("Error: File not found!");
System.err.print("Invalid input.");

Default destination: Console (often appears in red)

Key Points to Remember:

  • These streams are automatically available when your Java program starts
  • They can be redirected to other sources/destinations using System.setIn(), System.setOut(), and System.setErr()
  • System.out and System.err are both PrintStream objects
  • System.in is an InputStream object
  • Error messages sent to System.err are often displayed in red in consoles
